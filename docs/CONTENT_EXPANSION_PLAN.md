# Content Expansion Plan

This plan tracks the next product-depth upgrades for Praesidium. The goal is to move from a polished release candidate toward a stronger long-term product.

## Target content shape

| Area | Current direction | Strong product target |
| --- | --- | --- |
| Campaign levels | 6 themed levels | 10 to 12 levels |
| Boss variety | Boss warning and stronger feedback | 4 to 6 recognizable boss patterns |
| Enemy variety | Core enemy roster | More traits and counterplay moments |
| Towers | Core tower roster | 1 to 3 additional late-game choices |
| Retention | Daily missions and weekly mode metadata | Visible daily and weekly goals with rewards |
| Data feedback | Product Health tab | Tune balance using outside playtest data |

## Boss pattern backlog

### Shield Carrier

- Role: teaches shield-breaking and sustained damage.
- Counterplay: Tesla, focused upgrades, timing Meteor before shield recovers.
- Risk: should not appear before players understand damage types.

### Swarm Caller

- Role: creates pressure by pairing a large target with smaller support enemies.
- Counterplay: Mortar, Cannon, Frost, early area damage.
- Risk: can overwhelm mobile players if introduced too early.

### Overdrive Runner

- Role: creates a speed spike and tests path coverage.
- Counterplay: Frost, Tesla, front-loaded damage.
- Risk: should have clear warning text and readable visual trail.

### Siege Core

- Role: slow but durable late-game check.
- Counterplay: upgraded Sniper, Cannon, talent scaling.
- Risk: can become boring if it is only high HP; needs phase feedback.

## Campaign expansion sequence

1. Add two mid-campaign levels focused on area damage and slow/control.
2. Add one late-campaign level focused on split pressure.
3. Add one final campaign level with a distinct boss pattern.
4. Add balance simulation coverage for every new level.
5. Update Intel Codex with the new enemy and boss traits.

## Weekly mode integration targets

1. Show weekly mode on the main menu beside daily objectives.
2. Apply non-destructive modifiers only after balance testing.
3. Keep weekly mode optional until Normal campaign is stable.
4. Add one summary row to the end screen when a weekly run is active.

## Acceptance criteria

- No new level should rely on only one mandatory tower.
- Every new boss pattern must have at least two clear counters.
- The first three levels must remain beginner-friendly.
- Weekly mode must not replace the normal campaign path.
- Product Health risks should be reviewed after every outside test batch.
